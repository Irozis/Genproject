# Методология экспериментальной проверки

Документ фиксирует актуальную методику проверки программного комплекса для адаптации рекламных материалов к различным цифровым форматам. Экспериментальная часть используется для подтверждения технического эффекта: выявления critical-дефектов, снижения `actionsToFix` и формирования воспроизводимых отчетов.

Система формирует стартовую технически пригодную заготовку, а не финальный дизайнерский макет. Поэтому визуальная оценка проверяет восприятие подготовленных заготовок и наличие критических визуальных проблем, но не является полной художественной экспертизой дизайна.

## Цель проверки

Цель эксперимента - сравнить методы первичной технической адаптации рекламного материала к разным цифровым форматам и оценить, насколько перестроение компоновки снижает количество critical-дефектов и объем последующей доводки относительно простого масштабирования.

## Исходные данные

В эксперименте используется каталог из 126 рекламных форматов. Для каждого формата применяются три метода адаптации, поэтому автоматическая часть эксперимента содержит 378 случаев:

```text
126 форматов x 3 метода = 378 случаев
```

Сравниваются методы:

1. `scaling` - простое масштабирование исходной компоновки;
2. `fixedLayout` - детерминированное перестроение компоновки;
3. `candidateSelection` - выбор кандидата с FixedLayout fallback.

## Автоматические метрики

Для каждого случая формируется набор технических метрик:

- `criticalCount` - количество critical-дефектов, при которых заготовка требует обязательного исправления;
- technical score - интегральная техническая оценка результата;
- `actionsToFix` - оценка объема доводки, необходимой для исправления выявленных проблем;
- `estimatedCorrectionTimeSec` - расчетная оценка времени исправления.

Валидатор проверяет технические свойства результата: соответствие формату, читаемость текстовых блоков, наличие важных элементов, риски обрезки, safe-area и другие признаки технически непригодной заготовки.

## Pairwise-сравнение

Для оценки безопасной надстройки `candidateSelection` отдельно выполняется pairwise-сравнение с `fixedLayout`.

Фиксируются:

- сколько раз выбран `fixedFallback`;
- сколько раз выбран `generated`-кандидат;
- в скольких случаях `candidateSelection` лучше `fixedLayout` по `actionsToFix`;
- в скольких случаях результаты одинаковы;
- в скольких случаях `candidateSelection` хуже.

Актуальные результаты pairwise-сравнения:

- `fixedFallback` выбран 116 раз;
- `generated` выбран 10 раз;
- `candidateSelection` лучше `fixedLayout` по `actionsToFix` в 10 случаях;
- одинаково в 116 случаях;
- хуже в 0 случаях.

## Визуальная оценка

Визуальная оценка дополняет автоматические метрики и проверяет восприятие подготовленных технических заготовок. Она не доказывает художественное качество и не заменяет дизайнерскую экспертизу.

Параметры визуальной оценки:

- 14 респондентов;
- 60 PNG;
- 840 оценок;
- шкала 0-2.

Итоговые значения:

| Метод              | Средний балл | Критический брак |
| ------------------ | -----------: | ---------------: |
| scaling            |         1,44 |      31 / 11,1 % |
| fixedLayout        |         1,75 |       12 / 4,3 % |
| candidateSelection |         1,73 |        6 / 2,1 % |

## Итоговые автоматические результаты

| Метод              | Critical | actionsToFix | Visual score |
| ------------------ | -------: | -----------: | -----------: |
| scaling            |       43 |          157 |         1,44 |
| fixedLayout        |        0 |           42 |         1,75 |
| candidateSelection |        0 |           29 |         1,73 |

Основной эффект достигается при отказе от простого масштабирования и переходе к перестроению компоновки. `fixedLayout` устраняет critical-дефекты, выявленные у `scaling`, а `candidateSelection` использует `fixedLayout` как fallback и снижает `actionsToFix` в части случаев без ухудшения по этой метрике.

## Ограничения

- `textMeasure` не является полноценной моделью многострочного SVG-текста.
- Часть проверок safe-area имеет heuristic status и используется как инженерная эвристика.
- Система формирует техническую заготовку, а не финальный дизайнерский макет.
- Визуальная оценка проверяет восприятие заготовок, но не гарантирует художественную завершенность результата.
